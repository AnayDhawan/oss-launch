# launch-playbook.md — getting the first wave of attention

Ready-to-edit post templates live in `launch/` (`hn-post.md`, `reddit-post.md`,
`yt-script.md`).

## Show HN
- Post Tuesday-Thursday, 8-11 AM US Eastern.
- Title: `Show HN: {{REPO}} - {{one-line pitch with the differentiator}}`.
- Immediately post a self-comment: (1) why you built it, (2) the interesting technical
  parts, (3) what feedback you want.
- A live demo URL is mandatory; HN users will not clone to try.
- Never ask for stars in the post.

## Reddit
| Subreddit | Best for | Angle |
|-----------|----------|-------|
| r/SideProject | Any project, welcoming | Personal story + demo |
| r/programming | Dev tools, libraries | Problem + architecture |
| r/webdev | Web apps, full-stack | Show the UI, focus on UX |
| r/opensource | Any OSS project | Lead with "free", "self-hosted" |
- Post to 2-3 subreddits, spaced ~48h apart. Reply to every comment within 24h.

## YouTube (under 4 min for a tool demo)
1. Hook (0-8s): show the end result first. Not "today I'll be talking about".
2. Problem (8-30s): two sentences; make the viewer feel the pain.
3. Demo (30s-3min): screen recording of real usage. No slides.
4. Code reveal (optional, 1-2min): show the clean, simple implementation.
5. CTA (last 15s): "Link in the description; if this helped, star it on GitHub."

## Star growth
| Tactic | Impact | Effort |
|--------|--------|--------|
| Animated GIF in README | Very high | Medium |
| One-command install | Very high | Low |
| Submit to `awesome-*` lists | High | Low (one PR per list) |
| Tweet with GIF on launch day | High | Low |
| Answer relevant Stack Overflow questions | High | Ongoing |
| `good first issue` label on 3+ issues | Medium (contributors) | Low |
| Blog post (dev.to / Hashnode) | Medium | Medium |
| Reply to every issue within 48h | Medium (retention) | Ongoing |
