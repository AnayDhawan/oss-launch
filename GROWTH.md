# Growth

**Position:** Make any repo open-source ready in one prompt — scan, fill the gaps with
adapted (not boilerplate) files, then launch it. Audience: solo maintainers and indie devs.

## Where this stands

v1.0.0 (2026-07-08). Public since v0.4.0 (2026-06-21).

## Channel plan

**Soft channels (repeatable, low-key) — open now:**
- PR into [`awesome-claude-code`](https://github.com/hesreallyhim/awesome-claude-code) — one-line entry + link.
- Claude Code Discord showcase channel, r/ClaudeAI — short "built this, here's the demo" post.
- Tag/credit any tooling authors referenced (none vendored here, but note prior art where relevant).

**One-shot channels (fire exactly once) — reserved for after this v1.0.0 lands and gets a short bake-in period:**
- Show HN: "Show HN: oss-launch — one-prompt OSS-ready repo"
- r/programming, r/opensource
- One X/Twitter launch thread with the demo asset as the hook
- Optional: Product Hunt

## Metric + kill date

**≥50 stars within 3 weeks of the one-shot push, plus 2 cross-posts.**

If that bar isn't met: stop active promotion, keep the repo maintained, bank it as a
portfolio artifact. Don't keep spending unpaid marketing effort past the kill date.

## Rules

1. The demo is the traction engine, not the install channel — lead every post with the
   real `example/` output or the demo GIF, not a feature list.
2. Credit upstream/tooling authors when posting.
3. Time-boxed — this never eats IOAI/IBDP/school time. Off-budget sessions only.
4. One-shot channels fire once, not on every release.
